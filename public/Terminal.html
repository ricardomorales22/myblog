Command Line

Command Line for beginners

Posted by Ricardo Morales

This lesson will be on the foundations of Command Line.

In this lesson you will learn the basics of Command Line. This will be essential to your career as a developer, as you will be using Command Line a lot.

First, what is Command Line? Command Line is an interface in which you can type and execute text based on commands.

Basics of Command Line

Directories:

pwd | Display the path of current working Directory


cd | Change directory ( This can be used to find any folder/desktop/anything under any name. Example cd Desktop will take you into your Desktop.)

cd .. | Navigate to parent directory ( Can also use this command to go back to last directory.)

ls | List directory content( It list all the files in a specific sector of your computer.)

ls -la | List detailed directory content including hidden files ( Most hidden files should not be erased or changed in any way, there is a reason they are hidden.)

mkdir < file name > | Creates a new directory( Directory = File. Whatever you put between the < > will the name of your directory. )

Files:

rm < file name > | Delete < file name >

rm -r | Delete Directory

rm -f < file > add -r to delete a directory | Force deletes file ( Before running this command make sure you know what you're deleting because this command has the potential to delete your whole system in seconds.)

mv < old-file > < new-file > | Renames files

mv < file > < directory> | Moves file to directory ( Can possibly overwrite an existing directory if not done properly.)

cp < file > < directory > | Copys files to a new or existing directory. ( Can possibly overwrite an existing directory if not done properly.)
cp -r < directory 1 > < directory 2> | Copy content in directory 1 to directory 2.

touch < file > | Update file access and modification times and used to create a new file.

Search:

find < dir > -n name "< file >" | Finds all files named file inside the directory (folder). Use * to search for partial file names.

grep "< text >" < file > | Output all occurrences of< text> inside < file > (add -i for case-insensitivity)
grep -rl "< text >" < dir > | Search for all files containing < text > inside < dir >

Getting help from Command Line:

On the command line, help is always at hand: you can either type man < command >| or < command > --help to receive detailed documentation about the command in question.
