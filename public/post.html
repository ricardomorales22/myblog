Installing your text editor, Ruby, and Exercise number 1

Ruby for beginners

Posted by Ricardo Morales

In reading my blog I assume you want to learn how to code, and because of this I will be hard on you at times and I will lend a hand at other times. So lets start with getting your text editor and installing Ruby.

Go to http://www.barebones.com/products/textwrangler/ to get your text editor ready. This is crucial, as soon you'll be writing code. Textwrangler comes pretty much configured how you would want it so don't worry too much about it. We will tweak it along the way, if needed. Put textwrangler on your Dock and put the Command Line on your dock. If you don't know how to find your Command Line, Google it. I want you to learn to think like a programmer. Run your Command Line program. If you don't know how to do that, Google it or keep reading. Once you run it, it will look kind of empty, thats normal. In your Command Line, run irb (Interavtive Ruby). Things are ran in the Command Line by just typing their names and hitting enter.

If irb is not running (irb is not recognized) Install it using the RVM Ruby Version Manager. Again, Google it. To get out of irb simply hit control D. Hopefully, you see the same thing as you did before you ran irb, if you don't. Google it.

Now in your Command Line I want you to make a directory(file) and the way you do this is by typing this acroynm: mkdir and right after the file name. (example: mkdir rubyfile) and I want you to learn how to change directories and the way to do this is simply by putting this acroynm: cd (change directory) (example: cd rubyfolder) this tells your Command Line to change directory into whatever folder you specified. (Pro tip: if you type cd and the first letter of wherever you're trying to go and press tab, Command Line will autofill the rest for you)

Use your editor to create a file in this directory(folder). You will make the file, ‚ÄúSave‚Äù or ‚ÄúSave As...‚Äù, and pick this directory. Go back to the Command Line by just using your keyboard, you can do this by holding command and pressing tab at the same time. Back in the Command Line see if you can list the directory to see your new file. If you don't know how to do this, Google it. (Hint: Google this term "ls Command Line")

Exercises numbers 1 - 3

Exercise Number 1

If you are not comfortable with your text editor go back and practice saving and editing your file until you are.

To begin, the word "puts" tells the computer to print out whatever you put in the " " after the word puts.
puts "Hello World!"
puts "Hello Again, World!"
puts "I enjoy typing this."
puts "This is fun."
puts 'Look! I'm Printing.'
puts "I'd much rather you print this."
puts 'I "said" print this.'
Type this don't copy and paste it, type it and save it in your computer under rubyex1.rb (The .rb tells the computer this is a ruby file)
Then run the ruby file on your Command Line by typing

ruby rubyex1.rb

What your output should look like:

Assuming you did this correctly this should be your output:


Hello World!
Hello Again, World!
I enjoy typing this.
This is fun.
Look! I'm Printing.
I'd much rather you print this.
I "said" print this.


If your Command Line isn't printing this, you did something wrong. Go back and check what you did wrong, fix it and move on to exercise two.


If you have an error it will look like this:
ruby ex1.rb

ex1.rb:4: syntax error, unexpected tCONSTANT,
expecting $end

puts "This is fun."
^

Its imperative that you learn how to read these errors because you will see them over and over again so lets break this down line by line:
1. Here we ran our command in the Command Line to run the ex1.rb script.

2. Ruby then tells us that the file ex1.rb has an error on line 4.

3. It then prints this line for us.

4. Then it puts a ^ (caret) character to point at where the problem is.

5. Finally, it prints out a ‚Äúsyntax error‚Äù and tells us something about what might be the error. Usually these are very cryptic, but if you copy that text into a search engine, you will find someone else who‚Äôs had that error and you can probably figure out how to fix it.
